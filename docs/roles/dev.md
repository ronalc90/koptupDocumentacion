# DEV (Desarrollador)

## Propósito
- Implementa funcionalidades y mantiene la documentación técnica y funcional.

## Capacidades por módulo
- Documentos: crea/edita; sube adjuntos; comenta; mantiene versiones.
- Validación/QA: atiende incidencias; actualiza documentos hasta aprobación.
- Ágil: crea tareas; actualiza estados; vincula documentación.
- Checklist: completa ítems (documentación, historias cerradas).
- Auditoría: consulta accesos propios y eventos relevantes.

## Flujos clave
- Redacción y actualización de documentos vinculados a historias/tareas.
- Atención de feedback QA y resolución de incidencias.
- Gestión de tareas y estados en Ágil.

## Enlaces UI
- `/documents`, `/documents/:id`, `/agile`, `/validation`, `/checklist`

## Endpoints
- Ver índice general en `../index.md`.

## Nota
- El rol ADMIN puede acceder y utilizar todos los flujos para propósitos de prueba.
